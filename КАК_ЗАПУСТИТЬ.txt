TOURNEYFORGE — КАК ЗАПУСТИТЬ ПРОЕКТ
====================================

Что это
-------
Веб-платформа для организации турниров и LAN-вечеринок с RPG-карточками команд.
Учебный проект. Backend — Верещагин Сергей, frontend/дизайн — Герасимов Владимир.

Что нужно установить заранее
-----------------------------
Node.js версии 18 или новее: https://nodejs.org
(при установке ничего дополнительно настраивать не нужно, npm идёт вместе с Node.js)

Быстрый запуск
--------------
1. Дважды кликни ЗАПУСК.bat
2. При первом запуске откроется два окна консоли и автоматически:
   - установятся зависимости backend и frontend
   - создастся файл backend/.env со случайным JWT_SECRET
   - создастся база данных и зальются демо-команды
   Это может занять пару минут. При следующих запусках всё это уже будет готово,
   и оба сервера просто стартуют сразу.
3. Когда оба окна консоли покажут, что серверы запущены — открой в браузере:
   http://localhost:5173

Чтобы остановить — просто закрой оба открывшихся окна консоли (backend и frontend).

Структура проекта
-----------------
frontend/       — сайт (React + Vite), порт 5173
backend/        — сервер (Express + Prisma + Socket.io), порт 4000
admin-panel/    — отдельная админ-панель для управления данными без консоли БД, порт 5588
docs/           — техническая документация (спецификации по неделям)
README.md       — подробное техническое описание: архитектура, стек, модель данных, статус

Тестовые аккаунты
------------------
Администратор (может создавать/редактировать команды и турниры):
  логин: Admin
  пароль: admin

Обычный пользователь (только просмотр):
  логин: User
  пароль: user

Админ-панель (запускается отдельно, вручную)
----------------------------------------------
Открой отдельное окно консоли:
  cd admin-panel
  npm install
  node server.js
Затем открой в браузере: http://localhost:5588

Живая статистика игроков (необязательно)
------------------------------------------
Виджет живой статистики (FACEIT для CS2, HenrikDev для Valorant) работает только
если в backend/.env вписаны собственные ключи FACEIT_API_KEY / VALORANT_API_KEY
(как их получить — см. комментарии в backend/.env.example).
Без ключей всё остальное работает нормально, просто виджет статистики будет недоступен.

Если что-то не работает
------------------------
- Frontend работает даже без запущенного backend — просто показывает демо-данные
  вместо реальных, приложение не падает.
- Если после npm install что-то не запускается — проверь, что установлен Node.js
  версии 18+ (node -v в консоли).
- Подробности по архитектуре, стеку и статусу разработки по неделям — смотри README.md.
