# Генератор RPG-карток команд (PNG-експорт) — специфікація

На `/profile` (деталь-вигляд команди) вже є панель "Рідкість картки" з
захардкодженими бейджами (`Common`/`Rare`/`Epic`/`Legendary`, завжди
підсвічує `Epic`) — заготовка, яку ніхто не довів до кінця. Цей документ
описує, як зробити рідкість реальною і додати справжню RPG-картку команди з
експортом у PNG (пункт 12 календарного плану, Тиждень 4).

## Обсяг (scope)

- Рідкість рахується від **турнірних досягнень** (`team.tournaments` —
  лічильник виграних турнірів, вже існує й інкрементується/декрементується
  при завершенні/скасуванні фіналу), а не від рейтингу гравців. Причина:
  рейтинг команди в трьох дисциплінах виражений у несумісних одиницях
  (CS2 FACEIT ELO, Dota MMR, Valorant-звання) — чесне порівняння між ними
  неможливе без вигаданої нормалізації, якої проєкт свідомо уникає (див.
  рішення про відмову від єдиного вигаданого ELO). Кількість виграних
  турнірів — єдина метрика, однаково чесна для всіх дисциплін.
- Картка — окремий портретний макет під експорт, не знімок наявної панелі
  сайту. Показує лише основний склад (без запасних) і коротку статистику.
- Рендер у PNG — `html-to-image` (DOM → PNG), без Puppeteer/бекенд-рендерингу.

## Рідкість

Нове чисте обчислення (фронтенд, поруч із `avgRating`/`effectivePlayerRank`
у `frontend/src/lib/demo.js`), без змін на бекенді:

```js
export function teamRarity(tournamentsWon) {
  if (tournamentsWon >= 3) return "Legendary";
  if (tournamentsWon === 2) return "Epic";
  if (tournamentsWon === 1) return "Rare";
  return "Common";
}
```

Панель "Рідкість картки" на `/profile` перестає бути захардкодженою —
підсвічує тіер, порахований з `team.tournaments` реальної команди.

## Компонент картки

Новий `frontend/src/components/TeamCard.jsx` — портретний макет
(~600×840px), стилізований під наявну палітру "tactical cyberpunk"
(`void` фон, cyan/volt рамки, glow-фільтри — той самий патерн, що й
`Panel`/`Overline` в `components/arena.jsx`, не новий стиль з нуля):

- лого команди зверху (велике, `object-fit: cover`, як і скрізь на сайті)
- назва команди (Unbounded)
- бейдж дисципліни + бейдж рідкості (Common — сірий контур без glow,
  Rare — cyan контур, Epic — volt контур з легким glow, Legendary —
  посилений glow + тонка градієнтна рамка)
- велике число середнього рейтингу основного складу + одиниця виміру
  (перевикористовує вже наявний `avgRating`/`effectivePlayerRank`, ті самі
  дані, що на `/profile`)
- рядок короткої статистики: winrate / стрік / турніри / найкращий
  результат (ті самі поля, що вже показуються на `/profile`, фолбек "—"
  для порожніх)
- ростер основного складу: нік (live-підмінений через
  `liveNickFromStats`, як і в `PlayerRow`), роль, звання/рейтинг
  (live-підмінений через `liveRankFromStats`, як і в `PlayerRow`) — без
  запасних, без сортування, у тому ж порядку, що й масив `players`
- дрібний watermark-текст знизу ("TourneyForge" чи назва платформи)

Компонент отримує вже завантажені дані команди як пропси (той самий
`team`-обʼєкт, що вже є на сторінці `/profile`) — жодних нових API-запитів.

## Рендер і UX

- Залежність: `html-to-image` (додати в `frontend/package.json`).
- На `/profile`, у деталь-вигляді команди, кнопка "Картка команди" (поруч із
  наявним лінком "Редагувати") відкриває модалку.
- Модалка показує **живий превʼю** `TeamCard` як звичайний DOM (не PNG) —
  користувач бачить результат до збереження. Усередині — кнопка
  "Завантажити PNG".
- Клік на "Завантажити PNG" викликає `htmlToImage.toPng()` на DOM-вузлі
  картки, отриманий data URL підставляється в тимчасовий `<a download>` і
  клікається програмно — файл `<назва_команди>-card.png` завантажується в
  браузері.
- Лого команди — завжди `data:` URL (завантажене через власний
  crop-у-canvas флоу в `Team.jsx`), не зовнішнє посилання, тому CORS/
  canvas-tainting проблем для `html-to-image` не буде.
- Модалка перевикористовує вже наявний патерн `role="dialog"`/`aria-modal`/
  Escape-закриття з модалок рахунку матчу в `Tournament.jsx` (не новий
  примітив).

## i18n

Нові ключі (укр/eng) в `frontend/src/lib/i18n.jsx`: `profile.card.open`
("Картка команди"/"Team card"), `profile.card.download`
("Завантажити PNG"/"Download PNG"), `profile.card.rarity.<tier>` — назви
тіерів рідкості (лишаються англійськими словами `Common/Rare/Epic/
Legendary` навіть в укр-локалі, як і зараз — це усталені ігрові терміни,
не перекладаються по факту в жодній грі).

## Тестування

Ручна перевірка в браузері (не лише читання коду):

- Команда з `tournaments: 0` → рідкість Common.
- Тимчасово підняти `tournaments` тестовій команді напряму в `dev.db`
  (Prisma-скрипт, той самий підхід, що вже використовувався для локальних
  тестових правок цього тижня) до 1/2/3 → перевірити Rare/Epic/Legendary
  на панелі `/profile` і на самій картці.
- CS2-команда (числовий рейтинг) і Valorant-команда (текстове звання) —
  обидві коректно рендерять велике число/звання на картці.
- Команда із завантаженим фото-лого і команда з дефолтним плейсхолдером —
  обидва варіанти коректно потрапляють у PNG.
- Команда із запасними гравцями — перевірити, що запасні НЕ потрапляють на
  картку.
- Відкрити збережений PNG-файл і візуально звірити з превʼю в модалці
  (текст не обрізаний, лого не розтягнуте, рамки на місці).
