# Локальна admin-панель

Окремий локальний сайтик для керування вже задеплоєним на Render застосунком — без Shell/PowerShell для щоденних дій. Ходить напряму в той самий REST API, що й основний сайт (`https://tourneyforge-backend.onrender.com` за замовчуванням, змінюється в полі "Backend URL" у шапці).

## Запуск

Подвійний клік на `start.bat` — відкриє браузер на `http://localhost:5588` сам.

Або з терміналу: `node server.js` (з цієї теки) чи `npm start`.

## Вхід

Той самий логін/пароль, що й на основному сайті — потрібен акаунт з роллю `admin` (частина функцій доступна й `organizer`, але керування користувачами — тільки `admin`, як і на самому сайті).

## Що вміє

- **Користувачі** — зміна ролі (одразу зберігається), скидання пароля, видалення акаунта.
- **Команди** — ручна зміна рідкості картки, видалення. (Створення й повне редагування складу — на самому сайті, `/team/:id`, це складніша форма.)
- **Турніри** — створення, реєстрація команд, генерація сітки, зміна статусу, видалення. (Введення рахунку конкретного матчу — на сайті, у сітці турніру, там воно вже зручне.)

## Нюанс

Це просто UI-обгортка над існуючим API — власної бази даних чи бекенду не має. Якщо змінити URL бекенда на щось інше (наприклад, після редеплою під новою назвою) — введи новий у полі "Backend URL", збережеться в браузері.
